El Comportamiento Normal: Tipado Dinámico

Por naturaleza, Python es de tipado dinámico: una variable puede cambiar de tipo en cualquier momento.

variable = 10
print(type(variable)) # 

variable = "Hola"
print(type(variable)) #

¿Qué es el Tipado Estático?

La práctica de añadir "pistas" de tipo (Type Hints) a variables y funciones. Es opcional y no cambia el comportamiento del programa.

3 / 17

Beneficio 1: Claridad

El código se vuelve autodocumentado. Es mucho más fácil entender qué tipo de datos espera una función.

# ¿Qué es 'datos'? ¿Una lista, un diccionario?
def procesar(datos):
    pass

# Mucho más claro: espera un diccionario de string a entero
def procesar(datos: dict[str, int]):
    pass
4 / 17

Beneficio 2: Detección de Errores

Herramientas externas pueden analizar tu código y encontrar errores de tipo antes de que lo ejecutes.

5 / 17

Beneficio 3: Mejor Autocompletado

Tu editor de código (IDE) usará las pistas de tipo para ofrecerte mejores sugerencias y autocompletado, aumentando tu productividad.

Sintaxis: Variables

Se usa : tipo después del nombre de la variable.

nombre: str = "Ana"
edad: int = 30
es_estudiante: bool = True

Sintaxis: Funciones

Se anotan los argumentos con : tipo y el valor de retorno con -> tipo.

def saludar(nombre: str) -> str:
    return f"Hola, {nombre}"

def sumar(a: int, b: int) -> int:
    return a + b
8 / 17

Importante: Son Solo "Pistas"

El intérprete de Python ignora estas anotaciones. El código se ejecuta igual. Su propósito es para herramientas de análisis.

def sumar(a: int, b: int) -> int:
    return a + b

# Python no se queja y ejecuta esto sin error
resultado = sumar("hola", "mundo")
print(resultado) # 'holamundo'

El Verificador: mypy

`mypy` es la herramienta estándar para el análisis de tipos. Se instala con `pip` y se ejecuta desde la terminal.

pip install mypy
mypy mi_archivo.py

mypy en Acción

Si ejecutamos `mypy` sobre el ejemplo anterior, nos avisará del error.

sumar("hola", "mundo")

# Salida de mypy:

Tipos Complejos: Módulo typing

Para tipos más complejos como listas de strings o diccionarios con claves y valores específicos, se usa el módulo `typing`.

12 / 17

List y Dict

Para especificar el tipo de los elementos dentro de una colección.

from typing import List, Dict

nombres: List[str] = ["Ana", "Luis"]
puntuaciones: Dict[str, int] = {"Ana": 95, "Luis": 88}
13 / 17

Tuple

Permite especificar el tipo de cada elemento de una tupla de tamaño fijo.

from typing import Tuple

coordenada: Tuple[int, int] = (10, 20)
persona: Tuple[str, int, bool] = ("Ana", 30, True)
14 / 17

Optional

Indica que una variable puede ser de un tipo específico o puede ser `None`.

from typing import Optional

def buscar_usuario(id: int) -> Optional[str]:
    if id == 1:
        return "Usuario encontrado"
    else:
        return None
15 / 17

Union

Indica que una variable puede ser de uno de varios tipos posibles.

from typing import Union

# Un ID puede ser un número o un string
def procesar_id(id_usuario: Union[int, str]):
    pass

Conclusión

El tipado estático es una práctica moderna en Python que mejora drásticamente la robustez, legibilidad y mantenibilidad del código, especialmente en proyectos grandes y en equipo.
